Wave 1

Zap, a QR code generator, is Stratosphere Technologies newest application. A user makes an HTTP POST request to Zap containing information such as:

  • URL to be converted (ex. www.example.com)
  • Desired QR code color (ex. #000000)
  • Desired size of QR code image (ex. 512 × 512 px)
The user then receives an HTTP POST response containing a PNG of the desired QR code.

Zap is not yet deployed to the public. Skye, the project manager of Zap, is trying to figure out what to price each request at so the company can make a profit. She has performed a study where she recorded the usage of the Zap application by a pilot group of 50 users for a month.

While the pilot group had only 50 users, Skye expects that Zap will have around 1000 users when it is deployed to the public. To account for unexpected demand, she has created a synthetic user recording that simulates 5000 users based on the original recording.

Rain plans to use the synthetic user recording to think of a sufficient cloud architecture to recommend to the software team. He also wants to help Skye figure out what price to assign requests. Rain also happens to be the developer of Jetstream, a cloud simulation tool, that allows cloud architects to test out cloud ideas (his work on Jetstream was one of the main reasons Stratosphere hired him).

Based on his cursory analysis of the user recording, he thinks a virtual machine deployed on Cumulus Cloud, the world's leading cloud provider, will be able to handle the requests. He wants to test this out on Jetstream before making his recommendations to Skye and the software team.

Task

  1. Deploy a virtual machine on Jetstream
  2. Connect the virtual machine
