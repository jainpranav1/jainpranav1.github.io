Wave 1

Zap, a QR code generator, is Stratosphere Technologies newest application. A user makes an HTTP POST request to Zap containing information such as:

  • URL to be converted (ex. www.example.com)
  • Desired QR code color (ex. #000000)
  • Desired size of QR code image (ex. 512 × 512 px)
The user then received an HTTP POST response containing a PNG of the desired QR code.

Zap is not yet deployed to the public. Skye, the project manager of Zap, is trying to figure out what to price each request at so the company can make a profit. She has performed a study where she recorded the usage of the Zap application by 50 users over a month. Rain plans to use the user recording to think of a sufficient cloud architecture for Zap. He also wants to help Skye figure out what to price to assign requests.

Rain is the developer of Jetstream, a cloud simulation tool, that allows cloud architects to test out cloud ideas (his work on Jetstream was one of the main reasons Stratosphere hired him). Based on his cursory analysis of the user recording, he thinks a virtual machine deployed on Cumulus Cloud, the world's leading cloud provider, will be able to handle the requests.

Task

  1. Deploy a virtual machine on Jetstream
  2. Connect the virtual machine
